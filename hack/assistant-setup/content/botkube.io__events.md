#### Automation Features, Filtering Config

This release adds several new granular filter criteria (labels, annotations, resource name, event.reason, and event.message) which allow you to be even more specific when defining an event source.

In addition, we've added actions! Actions are specific commands that you can configure to run automatically any time a specified event is received (based on those new filter criteria).
